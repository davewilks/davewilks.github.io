Icon and Button Images

Summary:

Defines the use of icon and button images.

Full Description:

Icons and buttons with an image are implemented in Willis Towers Watson products in two ways:

  • As self-explanatory (usually) standalone controls, without screen text. For example:
  • As buttons with accompanying text. For example: , or

The following standards apply for each case.

Standalone Icon

Insert standalone icons inline before their associated tooltip text. (See xxxTBD in TCOMM Processes for information about capturing icons.)

Example:

  1. To remove an assignment, click (Delete).

Button Icon with Text

When documenting a page with a button/image combination, such as , do not capture the image in the text (see Exceptions below). Boldface the button text in a task step.

Example:

  1. On the Report List page, click Add.

NOTE EXCEPTION 1: For TRSS reference topics only, include icons inline after the button text (capture only icon).

Example: Click Add (no boldface in reference topics)

NOTE EXCEPTION 2: Include the icon if it distinguishes one UI element from another. For example, in TRSS there are two search UI elements on major grid pages for Jobs, Employees, and Structures. One search element has an icon and one does not. In this scenario, you would include the icon for all topic types.

Example: You can perform a simple search from any page within Talent|REWARD by clicking the Search icon in the Navigation header.
