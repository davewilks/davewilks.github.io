Definitions for Fields & Screen Elements

Summary:

Whether definitions of user interface elements are in a table, list, or expanding text, the purpose is to define the elements in a way that the user can clearly understand.

The MMoS covers our accepted style for standard screen elements.

Examples:

There are no specific rules about whether you start the definition with a verb or a noun. Instead, consider the item being defined and choose an appropriate noun or verb to start the sentence.

  1. Name or Description labels—These often do not actively do anything, so do not use a verb and consider a simple definitions such as the name of the particular thing. If you can say more about such a basic item, do.
    For example, if the name identifies a certain element in the application and is selected elsewhere, this is a good place to indicate that.
  2. Interactive UI elements—These usually perform specific tasks. Consider using an associated verb to begin those definitions:
  • Tables and grids—use verbs like "list"
  • Check boxes—use verbs like "indicate"
  • Drop-down lists—use verbs like "select"
  • Logic buttons (yes, no)—use verbs like "identifies whether" or "indicates"
  • Buttons (Save, Open, Find, etc.)—use appropriate verbs for the action. Save usually just saves with no further input. Use: Click Save. Find or Edit typically do something, so use something like "opens the edit window."
  • Tabs—use a more complete description in a task step, such as: Click the Snippet Conditions tab.
