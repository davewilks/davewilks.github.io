Writing Task Topics

Ten+ Commandments of Task Topics
  1. Task topics are less prescriptive, they should be logical and informative, not mechanical.
  2. Do share, in an introductory statement, all possible tasks. BTW – “manage” goes a long way with an accompanying image.
  3. Do compile obvious steps into a “Define the blah-blah” list, if warranted.
  4. You are not required to include screen shots. Screen shots should tell a story.
  5. You are not required to document everything. DO document the gotchas (User: uh-oh, oops, I didn’t know) in Notes or Tips.
  6. You are not required to document multiple ways to do things. The happy path is fine.
  7. You are not required to include result sentences or paragraphs. Use them when they help.
  8. You are not required to document the Cancel button in a common task.
  9. You can use Click Save without additional information. (“Click Save.” is better than “Click Save to save your changes.”
  10. You are not required to document how many characters are allowed in a field.
  11. Refer Links to other supporting documentation (e.g., reference topics, training).

Using Links in Task Topics

Hyperlinks—Use hyperlinks to other relevant topics in steps. There are formal and informal ways to reference the other topic. The first method is preferred:
- You must apply an alignment table to the jobs before applying a communicated level. Use this method unless awkward.
- See <title of topic> for more details.

Cross-reference links—Use this link type for PDF output. A page number is added to the topic cross-references, in the PDF. To accomplish this, set up the MadCap xref CSS as: {para}, {pageref}

Bookmarks—Use bookmarks when you want intra-topic links. Insert a named bookmark at the destination location in the file. Insert the Hyperlink to the Bookmark 'place in the file' at the jump location. See Properties example above.

Glossary —Set glossary term links to display in on-screen popup windows (this is the default in Flare). The popup appears when you hover over the term. See Glossaries in Working with Flare Processes.
