Targets

Summary:

Describes the use of targets within Iridize.

Full Description:

Iridize selects targets using jquery selectors. The Target Selector enables you to use the developer tools and select selectors via a jquery/CSS selector. Use the up-arrow on the interface to go up the DOM tree and automatically select the container of the existing target element.

These are custom Iridize selectors that we have added to the usual jQuery selectors. You can find the list of basic jQuery selectors here http://api.jquery.com/category/selectors/. To that list you may add the following custom Iridize selectors, each of which correspond to a jQuery DOM traversal method.

  1. The ">>" selector - this is equivalent to jquery.fn.find. For example, ".foo >> .bar" will look for an element with class "bar" contained within an element with class "foo". You may think of it as "look for elements .foo, then look within their child elements for .bar".
  2. The "<<" selector - this is equivalent to jquery.fn.parents. For example ".foo << .bar" will look for an element with class "bar" within the ancestor elements of an element with class "foo". You may think of it as "first look for elements with .foo, then within each element's ancestors look for .bar".
  3. The "!" selector - this is equivalent to jquery.fn.not. For example, ".foo ! .bar" will select elements with class "foo" if they do not also have the class "bar". You may think of it as "look for elements with .foo, then filter out all elements with .bar".
  4. The ":first" selector selects the first matched element. For example, ":first[title="Legend"]" will select the first element with the title "Legend". This may prove useful when tutorials reference the Dashboards where multiple analytics will be compiled into one window. (http://api.jquery.com/first-selector/)
  5. The ":last" selector selects the last matched element. For example, ":last[title="Legend"]" will select the last element with the title "Legend". (http://api.jquery.com/last-selector/)
  6. When a selector appears as a numbered element, such as [name="168"], the element number is likely to change and should not be used as a selector. In this situation, move up through the code until you locate a viable label and replace it. For example, we replaced [name="168"] with [ng-model="newScenario.Type"]:first.

NOTE Helpful Links

See http://api.jquery.com, http://www.w3schools.com/jquery/jquery_ref_selectors.asp, and http://www.w3schools.com/cssref/css_selectors.asp for selector ideas.
