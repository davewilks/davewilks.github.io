Editing Checklist for TMS Hotfix Release Notes

Use this list to check the HF RN drafts for writing style errors and inconsistencies. Refer to otherTech Comm Standards for editing draft content as well as this list.

Date Assignee Tasks
Tues (1st week) to Thurs (2nd week) TCOMM In the Summary, write just what is wrong with the software or what the user sees that is wrong. In the Resolution, first state what the user sees as correct (in the first sentence). Then state any configuration information.
Do NOT use the verb "choose". Use "select" or "click".
Use lowercase if you are discussing proper nouns (e.g., administrator, manager, primary reviewer) generically in a sentence.
Do NOT use the passive indefinite verb “is displayed” unless you need a direct object in the sentence. Use the active and intransitive verb "appears".
Format UI text in the same case used in the UI.
Do NOT hyphenate compound nouns (e.g., mid year, read only) if they do not modify another noun (i.e., act as an adjective).
Use "software" when referring to TMS or its software sub-components instead of system, application, or other similar words.
Use "employee" or “user” to refer to anyone in ESS with appropriate permissions. Use "administrator" to refer to someone who can make changes in the Admin side of TMS.
Highlight new features in the text.
Do NOT write terminologies with their acronyms (e.g., self-assessment (SA), employee evaluation (EE)) if they are not reused in the release notes. Note that self-assessment is hyphenated in the TMS UI.
Use "then" to show another action in the same view. Avoid colloquial, slang, jargon, and informal language. These types of words or phrases do not translate well to other languages.
Reuse content whenever possible, for example, a hotfix may require more work in a subsequent release, so refer to the previous write-up and reuse as much of the content as possible.
