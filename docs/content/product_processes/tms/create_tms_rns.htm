Create TMS Release Notes and Help Center Topics

Summary:

Describes how to manage the product release notes (RNs) and Help Center topics (HCTs) for the twice-yearly product release of the Willis Towers Watson Talent Management Software.

Full Description:

TComm uses the TRSS 8.# tickets and Agile sprints to develop the RNs and new/updated HCTs for the major product releases. TComm handles the minor product releases (RNs and HCTs) in the HF RNs process.

  • For the RNs, the Product Owners (POs) add a label (ReleaseNotes) to the sprint tickets to tag the content that should go into the RNs. You may have to remind the POs to tag the tickets. Several TComm writers work from the Agile 8.# sprint tickets to create the content (in narrative form), and update a ticket spreadsheet (in TRSS1.flprj > Release Notes > 8.#_RNs) to track the work. We use the RNs drafts to manage reviews with the PMs and POs. We use the TRSS1.flprj > Release_Notes folder to manage the versions and review. We post the final RNs on the TRSS Supporting Documentation page.
  • For the HCTs, the TComm writer adds a label (HelpTopic) to the sprint ticket to tag the content that should go into a HCT. The TComm writer then creates a TComm ticket to track the work to add a new topic or change an existing topic and the peer review. To conduct the functional (SME) review, the TComm writer creates a TRSS ticket for the next TMS sprint and manages any reviews in sub-tasks.

Tips

  • TComm has about 10-15, two-week sprints to finalize the RNs with the POs and do the final review with the POs and PMs. TMS sprints are every two weeks from Wed-Tues. TComm sprints are every two weeks+ from Wed-Fri.
  • TComm has one sprint (10), the last two weeks of the release, to finish the final reviews and changes.
  • Use text and tables in the RNs and, if absolutely necessary use images. In the HCTs (reference and task topics), use text and images.
  • You may not use every section in the RNs for a release: Career Management, Compensation Management, Learning Management, Performance Management, Reporting Management, Succession Management, and System Management. The Component field in the TRSS ticket tells you which section in the RNs to put the ticket in.

Sprint Sprint TComm/PO

For the HCTs:

  1. If the ticket content impacts the HCTs, add the HelpTopic label to the TRSS ticket.
  2. Create a new TCOMM ticket and add it to the next TCOMM sprint.
  3. Create the new content/update existing content in the topic.
  4. Use the TCOMM ticket for peer review.
  5. Create a Word file for the functional (SME) review.
  6. In the TRSS Help Review ticket, create a sub-task and link the TCOMM ticket.
  7. Use the sub-task to manage the SME review. Assign the sub-task to a PO and attach the Word file for review.
  8. When the reviews are complete, close the sub-task and close the TCOMM ticket.
Sprint Sprint TComm/PO

For the RNs:

  1. At the start of the release, create a new RNs Word file from either the previous one or the latest RNs template. Save in a new release folder in TRSS1.flprj > Release Notes >8.#_RNs .
  2. If a ticket will be in the RN, make sure there is a ReleaseNotes label in the TRSS ticket.
  3. In the RNs Word file, add the title and ticket number under the appropriate component section for reference. Remove the ticket number for the final draft.
  4. Create the content for the RNs section.
  5. Include the PMs and POs on all review emails.
  6. For the epics that are complete earlier in release process, coordinate with the TCOMM writers to send out a draft review of the RNs for the POs (Troy, Chot, Jeremy, 4 weeks before release, give them 1 week to review).
  7. Incorporate the POs review comments.
  8. For the final review, send out the RNs to the POs and PMs for final review (2 weeks before release, give them 1 week).
  9. There may be last-day changes. Coordinate with POs and PMs. In some cases, you may need to refresh the RNs final file up to one week after the release.

Figure process:

  • Add your images to the Images folder.
  • Snap images in ESS with the WTW color (was red). (ESS > Tools > Settings)
  • Sanitize the image data. Do not use test data or proper names or companies in the test data. If you need, use the Chrome > Inspect Element function to change data.
  • The figure style doesn't work by just selecting the figure and clicking Figure.
    • Insert your image in the Word document.
    • Apply the Figure style.
    • Select another figure with a shadow.
    • Click the Format Painter.
    • Click your image.
    • Add a caption below the image.
