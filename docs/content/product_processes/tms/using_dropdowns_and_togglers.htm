Using Flare Drop-downs and Togglers

Summary:

Describes the use of drop-downs and togglers and the steps to create each in Flare.

Full Description:

These two features serve similar purposes–hiding text under "hotspots" to let readers either choose sections of a topic they're interested in or choose whether to view or skip tangential information, particularly in lists. The features also make for a lean clean look and feel.

Creating Drop-downs

To create a drop-down, select the heading or step/list stem you want to use as a hot spot along with all the text you want to hide underneath ("drop-down body"), then Insert-->Drop-Down Text. See Flare Help for more info.

NOTE For nested drop-downs, create the deepest levels first, then higher levels; if you need to add a nested drop-down into an existing one, you must first unbind the existing one.

NOTE Sometimes the Flare dialog does not properly capture the drop-down body under the hotspot. When this happens, you can define the hotspot paragraph only and then manually move text blocks into the drop-down body.

NOTE KEYBOARD SHORTCUT. To quickly move text up or into a previous XML element (when allowed), select the block in left-side structure view, then use SHIFT-ALT Up-Arrow. (Use the down-arrow to move text down or out of an XML container, such as drop-down body.

Creating Togglers

Unlike drop-downs, where the hotspot is a heading or list stem, the toggler feature provides a way to use shorter text strings as hotspots. This feature is useful in task steps, for example, that result in application popups and/or lists of optional sub-steps or nice-to-know (or self-explanatory) definitions.

Solution:

Consider using togglers to successively hide long and/or deeply nested lists. The technique is also a recommended way to avoid inserting inline screen shots into expanded text in lists, which creates spacing problems.

Examples:

Creating Togglers

To create a toggler in Flare:

  1. Right click the block containing the text or image that you want readers to toggle on and off (hide/show).

    2. Use the left-side block layout. (If it's not there, click the "Toggle show blocks" icon at bottom of the screen.)

    Alternatively, you can quick-click text repeatedly until the XML element you want is highlighted.

    1. Select Name in right-click menu.
    2. Give the XML element a name (no spaces).

      3. Ensure the name is unique for the topic.

    TIPYou can name any XML block in Flare, including snippet blocks; for example, hiding single-sourced field-description lists under task steps helps avoid extra scrolling one step to next.

  2. Repeat previous step for any surrounding blocks you want to control using the same toggler.
  3. Select text you want to use as a hot spot above the named targets and click Toggler in the Insert menu tab.
  4. Highlight only enough text to describe the hidden blocks clearly. Revise if needed to get a clear word grouping.
  5. In the Insert Toggler dialog, select the Named Elements you want the hot spot to control.
  6. Click OK.
  7. Preview the topic and test.

Toggler and Popup Tips

  • You can’t toggle information from another file.
  • Popups don't work in snippets.
