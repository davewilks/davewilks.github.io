Using File Tags

Summary:

Describes the file tags that are used within the TRSS project and the process of adding a tag.

Full Description:

TRSS projects use file tags to track information at a file level.

There are three sets, or categories, of tags:

  • Author - Indicates which team writer is writing the topic.
  • Status - Indicates the stage of development for the topic and whether there are outstanding graphics tasks (Needs Screen Shots) and linking tasks (Needs Xrefs).
  • Release - Shows the scheduled TRSS release version for the topic.

Solution:

To set tags in a topic:

  1. Open the topic.
  2. In the topic Properties dialog box, click File Tags.
  3. On the File Tags menu, click Author and select your name.
  4. Click Status from the menu and select a status. (Optional) Select Needs Screen Shots and/or Needs Xrefs if you want to track pending work items.
  5. Click Release from the menu, and click a release version, if listed. If the topic is not yet scheduled for a release, use TBD.

NOTE Setting tags at a folder level

You can add and remove file tags at a folder level while leaving the existing tags in place. For example, if you want to add a Release tag to all files in a folder, right click the folder and click Properties to open the Properties dialog box. (The tags at the folder level display blue check boxes indicating varied settings among the files in the folder.) You can leave those settings alone, or change them. If you add or remove one tag, that tag is changed for all files in the folder, but the other tags are unaffected.

Viewing a list of files with a specific tag

To see a list of files that have a specific tag in Flare, open the File List, scroll to the right to locate the column heading for the tag type and click the heading to sort the column.

You can also create reports with file tag filters. For more information about reports, see Flare help.
