Pre-release Check List for TMS

Summary:

TComm uses a pre-release checklist in the two weeks prior to a release to perform periodic tasks such as spell checks and release-specific tasks such as checking that a particular topic is conditionalized correctly for release. Throughout the release, any writer can add items to the latest sheet in the pre-release checklist. For example, for the 8.5 sheet, a writer can add that the product name changed from TRSS to Willis Towers Watson Talent Management Software and the variables need to be changed.

Full Description:

The pre-release checklist uses color coding for tasks that the writers perform every release and other tasks that are one-time only changes specific to a release:

  • URL: pre-release checklist. Right-click this link to open in a new tab. It gets messy if you open it in the topic.
  • Log in: Mark Cowan shares the document in Google Docs. Use the Sign In button in the upper right of the sheet to log-in with a google account.
  • Add Tasks: Go to the bottom of the latest sheet and add a task. Color code it if necessary.
  • Reviews: Two-weeks before the release we will have a pre-release checklist meeting and assign the tasks and reviews.
