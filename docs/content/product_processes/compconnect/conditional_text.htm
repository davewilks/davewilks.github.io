Conditional Text for Roles

Summary:

Compensation Software has some complex conditional text needs to provide help based on the user's role.

Full Description:

Compensation Software includes 3 targets to publish help for users who have the Global Grading feature set ( Role_GGS), the Market Analysis feature set (Role_MA), or both (Role_MA_and_GGS). Each target includes a specific set of conditional text roles to generate the appropriate text. Click the image to see which conditions are included or excluded.

The descriptions of the different Role conditions are included in the Flare project. The examples below supplement that information:

  • Using the GGS and Market_Analysis conditions: These conditions should cover 90% of all cases.

    • If an entire topic applies to only one role, conditionalize it at the topic level using either the GGS or Market_Analysis condition. This is the only circumstance where you should apply a condition at the target level.
      See Jobs > Job Leveling > Assigning_a_Business_Unit_to_One_Job.

    • If a paragraph or a larger chunk of content applies to only one role, conditionalize the content block.
      See Jobs > Managing Jobs > Adding_and_Copying_Jobs.
    • If content within a paragraph applies to only one role, DO NOT automatically conditionalize it. Consider whether you should or should not conditionalize. Most of the time the answer is no, it is too granular.
      See Jobs > Managing Jobs > Deleting_a_Job and Jobs > Managing Jobs > Adding_and_Copying_Jobs.

    • If a warning is related to functionality not available to one role, conditionalize it.
      See Jobs > Managing Jobs > Adding_and_Copying_Jobs.

  • Using the GGS_Only, MA_Only, and GGS_and_MA conditions: If a Topic contains content that meets all three of the conditions listed below, you can use these conditions. You must use all three; if you only need two, use the main GGS or Market_Analysis conditions. See the Getting Started topic for an example.
    • The topic contains content only for GGS users.
    • The topic contains content only for Market_Analysis users.
    • The topic contains content only for users who have both GGS and Market_Analysis.

A few more rules about the GGS_Only, MA_Only, and GGS_and_MA conditions.

  • Never use GGS in the same topic that has GGS_Only; same for Market_Analysis content.
  • Do not use topic-level conditions on topics with these conditions.
  • Use all three in a topic. Otherwise, you don't need to use them at all.

NOTE Do not put conditional text in Expanding Text as Flare embeds an additional expanding body element (with text only and an empty head tag) and this causes errors in the build process.

Check your work! Preview the topic using the appropriate target to see if a topic displays what you think it should.
