Translating the Compensation Software UI Text

Summary:

TComm is responsible for translations in the Compensation Software UI. You must have the Technical Communication Administrator role assigned to you to complete this work.

Full Description:

The basic process is to download the UI text from Compensation Software, send it to a translator, and upload it.

You need to log your week so we can keep track of what environment has been translated and when.

To translate UI text:

Note, steps 1-3 might be overkill.

  1. Create a ticket for the translation work. See this example.
  2. Use IE to open the Translations Confluence page. With IE you can edit the spreadsheet from the page. In browsers other than IE, access the Attachments and work with it there.
  3. Open the translation log and keep it updated throughout this process.
  4. On your machine, create a folder associated with a specific release. For example, I have a Translations folder with subfolders for R1, R1.1, and R1.2.
  5. In Compensation Software, navigate to TW Admin and access the Bulk Translations page (Standards > Bulk Translations).
  6. Download each of the resources to the release folder that you just created. Consider naming the zip file something descriptive of the resource, such as UI_Text.
    • Resources in Selected Language: select the language
    • Resources in GGS

      • NOTE The only GGS resources you need to translate are 3.2, 4.1, and 4.2.

    • Pay Elements
    • Standard Properties
  7. Extract the appropriate xlifs and rename each of them after you extract them. This is so you know which is which, plus they all have the same name which causes conflicts.
  8. Zip the files into one zip file, and then send the files to the translator. For EN-GB, send to Mark Cowan.
  9. After you receive the translated files, create a directory for them (mine is just called Translated) and save them there.
  10. For EN-GB, check the encoding to be sure it will work with the application:
    1. Open the file in Notepad++.
    2. Click Encoding, and make sure it is UCS-2 LE BOM.
    3. If not, select it and save the file.
  11. In Compensation Software, go to Bulk Translations and upload the files one at a time.
  12. Click Apply Translations after each upload.
  13. Attach your uploaded files to the JIRA ticket for safekeeping.
  14. Set your language, and then smoke test and report any bugs.
    1. Admin Home > Aging Percentages
    2. Admin Home > Employee Pay Elements

To smoke test:

Resources in English (main UI text)

  • Get ideas from the changed lines file.
    • Aging
      • TA>Aging Date> Confirm Ageing Dates in title and in text page.
      • TA>Survey Aging Percentages> Confirm Survey Ageing Percentages and ageing in text.
    • Leveling
      • Jobs> Select>Actions> Confirm Levelling
    • Annualize
      • TA>Pay Frequenices>Confirm Multiplier to Annualise
    • Organization
      • Jobs > Job Levelling > Organization Level (part of GGS?)
    • Admin Home > Employee Pay Elements

More notes:

If using Lingo and a TM, check the files for breaks. Search for </body in the middle of the file.

Use this regex to find and replace any existing translations: <target>(.*)
