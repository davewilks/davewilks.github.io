Tip Content Development

Summary:

Description of tips used in CompConnect and the process to write tips.

Full Description:

Tips in CoCo provide users with additional information about using the software. Tips include content that provides information such as non-obvious UI interactions to help prevent users from entering error conditions that could be avoided if they knew about it beforehand or insights into workflow including recommendations, shortcuts, and options that are not part of the core workflow.

To develop tip content:

  • Use the information that you obtain for the topics and checklists to put together an initial set of suggested tips for the functionality.
  • Email the Product Owner (PO) and ask to discuss tips for the feature:
    • Provide the initial tip suggestions that you create
    • Ask if the PO wants to either talk on the phone about the tips or simply edit what you provide and suggest any other tips she feels should be included based on her industry knowledge.
  • Refer to the Tips section of the CompConnect Standards page for more information on the Tip creation standards.

Solution:

Writing Tips in Flare

  1. In the appropriate subfolder of the Tips folder, create a new tip topic:
    1. Use the Tip topic template when you create tips:
      • Right-click the folder where you want to store the tip.
      • Select New | Topic | Tip_Topic_Latest.htm from the Templates folder.
    2. Leave the 1st Heading Box empty.
    3. Enter a human-readable and sensible Title, which appears in the topic. (Note: Need to determine an acceptable character limit.)
    4. Click Add.
    5. Delete the H1 text in the new topic.
  2. View the tip in the Text Editor in Flare and locate following line:

    3. <div id="identifier_01" class="tip">

  3. Replace "identifier_01" with a unique ID.

    4. To create an ID

    1. Open the application.
    2. View the source code (in Chrome, right-click the page> select Inspect Element; in IE, right-click the page> select View Source.
    3. Look for an ID in the code, common ones include a data ID.
    4. If there is no ID, name it based on the following convention: section_Heading_field_#.
    5. Update the Tip Tracking Spreadsheet in SharePoint.
  4. Add the stand-alone tip topic to the Tips TOC.
  5. Build the Tip target.

    6. The Tip Target applies the Tip Master Page that inserts the content in the following structure:

    Tip structure

    <div id="tip">

    <div id="title"><MadCap:variable name="System.Title" />

    </div>

    <div id="author">Towers Watson</div>

    <div id="content">

    <MadCap:bodyProxy />

    </div>

    </div>

  6. Perform a smoke test on local copy of CompConnect.
  7. Publish the tip target.
    1. In the CompConnect project, open the Tips target, and then click Publish.
    2. After the publication completes, open the CompConnect - TW Admin project.
    3. Open the Tips target, and then click Build.
    4. After it builds, naviagte to the output directoru on your machine (such as, C:\Workspace\CompConnect - TW Admin\Output\DAVE5957\Tips\Content\Tips).
    5. Copy all of the tips in the directory and paste them in the TW Admin directory at the published location:

      6. \\TRCIIS-NA2-201D\Design\help\CoCo_Tips\Content\Tips\TW Admin

  8. Add the tip information to the tip ticket for the development sprint.
