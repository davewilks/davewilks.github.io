JAMA Review Workflow <is this obsolete or does it need updating?>

Summary:

The Wasabi Tech Comm team uses JAMA to manage content reviews. Below is the process to set up and complete JAMA reviews.

JIRA Workflow Prep for a JAMA Review
  1. Check in all files.
  2. Add a date to the Due Date field in JIRA that is 3 weeks from the date you publish the JAMA review.
  3. Build the help to the review site:
    1. Add the topics to the appropriate TOCs (Wasabi Help and WAS_Tips). Do not remove any of the files already in the TOCs.
    2. Open the appropriate batch review target:
      • Tips and Topics only: Review Site Tips and Topics
      • Tips, Topics, and Checklists: Review Site Tips Topics and Checklists
    3. Click Build and Publish.
  4. After publication, complete the steps in Prepare the Flare Content for JAMA.
Prepare Flare Content for JAMA
  1. Open the Word_Review TOC in the Project Organizer.

  2. Delete the existing topics in the TOC, except the Tips topic which serves as a container for tips.
  3. Add the topics that you want reviewed.
  4. Add tip topics under the Tips book in the TOC.
  5. Save your changes.
  6. Open the Word_Review target.

  7. Click Build or View.
  8. After the build process finishes, the "Output was generated but with a set of warnings, listed in the "Warnings" section" message appears. Click OK.
  9. Click View Output.
Modify the Word Output

Add links to the JIRA tickets, Change the H1 styled paragraphs to Heading1, and identify approved content.

  1. Add links to the JIRA tickets after the first topic.
  2. Add a link to the review site after the first topic: http://trciis-na2-100d:82/help/Wasabi/default.htm.
  3. Open the Styles panel.

  4. With your cursor in the first heading, which should be shown as the h1 style, hover over the h1 style in the list and then click the down arrow to expand the Style menu.

  5. Click Select All.

  6. Click the Heading 1 style to update all of the paragraphs currently tagged with the H1 style to Heading 1.
  7. In the Tips section, add titles to each tip. Use Heading 2.
  8. Locate any Approved tips or other reused content, and then change the text color to green.
  9. Perform a quick search for any odd formatting.
  10. Save and close the document.
Import the Content into JAMA

In JAMA, create a set of texts and then import the Word Review document.

  1. Log on to JAMA and navigate to the TechComm project.
  2. Right click Wasabi Reviews or TRSS Reviews, and then select Add > New Item > Set of > Texts.

  3. Leave the Set Key as the default (TXT), but change the Name to something that covers the functionality, such as the name of the content folder in Flare.
  4. Click Save and Close.
  5. On the Comment and Notify Set window, click Commit.
  6. At the top of the navigation area, click Project > Import > Import.

  7. On the Select Import File and Destination window, click Choose File, and then select the Word file that you published from Flare.
  8. Click the Destination box, and then navigate to the Set of Texts that you created earlier.

  9. Click Select & Close.
  10. Click Next.
  11. On each window until the Final Import Summary window, click Next.

    12. NOTE On the How to Handle Tables window, make sure that Descriptions may include tables is already selected

  12. On the Final Import Summary window, click Close.
  13. Scan the imported files for any funky formatting. In particular, look for:
    • Words in bold with spaces removed.
    • Content that goes beyond the width of the screen.
  14. Right-click the Set of Texts, and then select Send for Review.
Manage Reviews

Managing issues in JAMA is a multi-step process. In general, it involves sending the initial review, responding to comments, editing the content in JAMA, indicating changes, and creating a new revision to either respond to more comments or move it along the process.

Create a JAMA Review
  1. On the Create review page, update the Review Deadline.
  2. Click Next.
  3. Select the reviewers. You might need to click Outside Project Team to locate the reviewers.
  4. Click Next.
  5. Customize the invitation, and then click Begin Review.
Respond to Comments and Make Edits

You can open the review at any time to see comments. However, to make sure that you aren't working in the review as the same time as other reviewers, wait until you receive a notification from JAMA that the review is completed.

The steps described in this section can be completed in any order depending on how you manage editorial comments.

  1. Open the review using any of the following methods:
    • Open JAMA, click REVIEWS, and then select the review.
    • Click the link in any of the emails you receive from JAMA.
  2. Respond to each comment:
    • Reply to open a conversation with your reviewer. This could be for something you disagree with or to ask a question.
    • Agree if you agree with the edit and will make the change.
    • Acknowledge if you agree with the edit, but need more information or want to indicate that your change might be different than what they expect.
  3. Complete your edits.

    4. NOTE A useful workflow might be to complete the edits in Flare with Track Changes on so that you don't miss any additions and deletions. Plus, you'll need to get this content into Flare anyway.

    1. In a section with comments, click the down arrow, and then select Edit Item.

    2. Edit the content:
      • Deleted text highlight in red and change the font to strikethrough.
      • Added text highlight in pale turquoise.
    3. Clear the Notify check box.
    4. Click Save and Close.
  4. After you respond to each comment and complete all of your edits, click Publish Updated Revision at the top of the page. See Creating JAMA Reviews for the steps.

    5. NOTE If this is a second revision or later, remove any highlights from previous revisions, add the phrase "Removed Highlighting," and then highlight it in yellow.

    If you made no changes to the section, type the phrase "No Changes," and then highlight it in yellow.

Implement JAMA Edits in Flare

You can copy and paste edits into Flare, keep up with the edits in Flare as you work through the process, or whatever process that you want to take.

After the reviewers approve topics and tips, you need to go into Flare and ensure that all changes are transferred there. In addition you need to indicate that the content is approved so that reviewers do not need to look at it again if it appears in future reviews as reused content.

To indicate that content is approved, we use the APPROVED snippet, which is conditionalized as "Comment" and should appear only in the Word Review target.

  1. Open the topic or snippet.
  2. Add the APPROVED snippet (Resources>Snippets) immediately above the approved content.
  3. Save the file.
