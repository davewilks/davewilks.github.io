Glossaries

Summary:

Use the glossary feature for terms that are esoteric industry concepts or internal Towers Watson jargon.

Full Description:

This section describes how to create a glossary term and set up variant glossaries.

When working with glossary terms, note the following information:

  • You can create auxiliary glossaries to use exclusively for inline hover popups. For example, extra glossaries can be useful for UI element definitions. You can exclude these glossaries from the build to prevent the terms from appearing in the output Glossary navigation tab.
  • Currently, Flare has a known limitation when using variants (for example, plurals) in the primary glossary: there is no way to exclude them from the navigation-tab list. You must use a supplemental glossary for variations of glossary terms (see Using a Variants Glossary). Future Flare versions may provide a way to specify "master" terms among listed variants in the same glossary, in order to exclude non-master variations from the output Glossary list.

Solution:

Create a Glossary Term

To create a glossary term:

  1. Open the Glossary in the Project Organizer:
    • TRSS1 - TRSS Glossary
    • CompConnect - Glossary
  2. In the Glossary editor, click New Item.
  3. In the Terms box, enter the glossary term.
  4. Enter the definition of the term in the Definition box.
  5. Click OK, and then save the Glossary file.
  6. (Optional) Add anticipated variants of the term in a separate glossary (see Using a Variants Glossary).

    7. NOTE You can also create glossary items on the fly. Just type the word, highlight it, and select Insert |Glossary Term Link. Fill in the definition and choose the glossary where the term should be listed.

  7. In Flare, highlight the first instance of the glossary term in the text (excluding the title) of your topic, and then click Glossary Term Link on the Insert ribbon.

Using a Variants Glossary

For the current version of Flare (10.x, 11.x), use a separate glossary for syntactic variations on terms in the main glossary. Variations should account for capitalized and plural forms of the word. For example, for the term aging date, add aging date, Aging date, aging dates, and Aging dates.

You can either add variants in advance or add them as you need them (if you know that the root term exists in the main glossary used for the build).

To add a variant glossary term:

  1. Open the variant glossary in the Project Organizer:
    • TRSS1- Variants
    • CompConnect - VariantGlossary
  2. In the Glossary editor, click New Item.
  3. In the Terms box, enter the variant glossary term(s). Enter multiple entries on new lines.
  4. If the entries are restricted to plural/singular or capitalization variations, leave the Definition box blank.

    5. Other variations, such as shortened terms, require definitions copied from the primary Glossary. This is not recommendedâ€“better to use both variations in the main glossary entry if it is reasonable for both terms to appear in the output Navigation panel, or rephrase the content.

  5. Click OK, save the glossary, and then check it in.

    6. NOTE Hovering over the glossary-linked item in the source file shows "def = " (no definition). However, Flare presents the root-term definition, in the main glossary, in the hover popup in output (built user view). Note that the term does not indicate the hover link in the quick topic preview.

To add variants on the fly:

  1. Enter the word in the topic file, and then highlight it.
  2. From the Insert menu, select Glossary Term Link. If the word is not already in a glossary, an entry dialog box appears.
  3. Select the variants glossary file (leave the Definition box blank for plural/singular and case variations).
  4. Click OK.
