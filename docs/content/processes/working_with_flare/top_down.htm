HTML5 Top Navigation Workarounds

Summary:

Flare's HTML 5 Top Navigation provides a more modern and responsive design. It is used for CompConnect help and other applications as well. This topic covers some of the wrokarounds needed to make Top Navigation work.

Full Description:

Poor Wrapping Around the Sidebar

Take a look at these two images:

The one on the left shows some poor wrapping around the sidebar. This typically happens with drop downs and tables.

To fix this, you just need to wrap the offending element in a div that has a setting like the one shown below in CompConnect:

Div.sideMenufix

{

overflow-x:auto;

}

In CompConnect, this Div is called sideMenuFix. To apply it:

Select Div.sideMenuFix from the Style box, or go into the XML and add <Div class="sideMenufix"> right before the element. Then add </Div> right after the element.

Adding Text to the Header

You might be asked to add a title, or other text, to the header area of the help. This isn't easy to do in Flare, but you can hack it using a script in the master page and some styles in the CSS.

To add a title the header area of top navigation:

  1. Set up the CSS styles that you'll need: a Div and a p class.

    2. Here's what is set up in CoCo:

  2. Open any master pages used in the help and add the following code just after the <body> tag:

    3. <script type="text/javascript">

    $(document).ready(function(){

    // add div for header text

    $("a.logo").after('<div class="title"><p class="helpTitle">Compensation Software Help Center</p></div>');

    });

    </script>

  3. Update the text (Compensation Software Help Center) to whatever you want to appear in the header.
  4. Update the div class and p. class to whatever you created.

Applying Different Master Pages to Different Topics

In many cases, you will want a home page that looks different than other pages. At the very least, you'll want to get rid of the menu on the right. You might want to have different master pages for different types of docs.

To apply different master pages:

  1. Create your alternative master page.
  2. Set up a new html class in the CSS to match your type of master page.

    3. In CompConnect, it looks like this and is used as a landing page, thus the name.:

    html.Landing

    {

    mc-master-page: url('../MasterPages/HomePage.flmsp');

    }

  3. Update the URL with the location of your alternative master page.
  4. Go to a topic that you want to use the alternative master page.
  5. Right-click the topic in the Content Explorer, and then select Properties.
  6. Go to the Topic Properties tab.
  7. From the Topic Style Class list, select the style you created in the CSS (Landing in this example).
