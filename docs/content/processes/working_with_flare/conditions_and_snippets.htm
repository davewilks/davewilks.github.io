Setting Conditional Text and Snippet Conditions

Summary:

To set conditional text and snippet conditions in files and on builds.

Full Description:

  1. Check out and lock a Flare .htm file.
  2. Create conditions, for example, Exclude in Build.
  3. For discrete text blocks: right-click the HTML blocks or chunks of text and select Conditions.
  4. For snippets, open the snippet and right-click the HTML blocks or chunks of text and select Conditions.
  5. Set the snippet conditions in the file.
    • Go to Content Explorer.
    • Check out and lock the .htm file.
    • Right-click the file in the explorer tree and select Properties > Snippet Conditions.
    • Select the conditions to include/exclude.
    • Check in the file.
    • DO NOT set the conditional text here. Set that in the next step in the target file.
  6. Set the conditional text in the target file.
    • Go to Project Organizer > Targets > [Build_Name].
    • Check out and lock the .htm file.
    • Double-click the file and select Conditional Text. (do not use right-click Conditions as I couldn't get that to work correctly. May be a Flare bug.
    • Select the conditions to include/exclude.
    • Check in the file.
    • Remember to set the conditions on all of the build files (reviews, QA, production).
  7. Test the builds to make sure the content is displaying correctly.
