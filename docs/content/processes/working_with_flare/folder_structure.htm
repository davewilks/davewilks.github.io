Creating Folder Structures

Summary:

For new TComm projects, use the following guidelines for folder structures in Flare. Given the variety of projects, you may or may not use all areas of the structure. Please add examples and variations as they occur (for group use).

Full Description:

Base the project's folder structure on logical functional folders and subfolders. The help (.htm) files will always be under a folder or subfolder. If you have images and snippets to use project wide, then put them at a higher level; however, if there are a large volume of images and snippets you may want to include them in functional folders and subfolders.

Sample 1: Generic Folder Structure Sample 2

Your mileage may vary. Use the names in bold as they appear below. No need to add the project name with, say, the Images or Snippets folders.

  • Content
    • Level 1: logical functional area
      • Level 2: related subfolder
        • Level 3: related subfolder
        • Level 3: Images [related area]
        • Level 3: help_topic.htm files
      • Level 2: Images [area related]
      • Level 2: Snippets [area related]
      • Level 2: help_topic.htm files
    • Level 1: Images [global project]
    • Level 1: Resources
      • Level 2: Images [formatting, skins, layout, master pages]
      • Level 2: MaterPages
      • Level 2: Snippets [global project]
      • Level 2: etc

NOTE The folder structure is based on Flare's advanced tri-pane layout.

Please add other structures as you create them.
