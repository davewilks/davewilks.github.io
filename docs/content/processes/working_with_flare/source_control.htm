Understanding Flare Source Control

Summary:

Flare Source Control allows for multiple authors to check out and work in the same file. Use these guidelines to avoid conflicts and collisions.

Full Description:

  • Do not assume you have the latest version of the files. Click Get Latest Version for each file before you open it. Flare does not automatically update the source control tree.
  • Pay attention to the Out of Date () and Checked Out to Other User () icons.
  • Use the Flare's Source Control reference for more information.
  • Set the Source Control options in Flare: File | Options | Source Control.

Source Control Icons

Status Description
Checked out The file is checked out from source control.
New file You have a new file in your project, but you have not yet added it to the project source control.
Checked in Not checked out, Checked in. The file is in source control, and is not currently checked out, which means that it contains a "Read Only" designation in its properties.
Checked out to other user The file is currently checked out to another user.
Out of date The file is not current (the local copy of the file is older than the source control copy).
Deleted The file is deleted from source control, but you still have a copy of the file on your local drive.
